# 01b — Training pipelines for LLMs and VLMs

## Why this matters

Every reading-list paper assumes you know the pre-train → SFT → preference-alignment stack. Without it, phrases like "instruction-tuned", "aligned with DPO", or "behavior-aware reward" are noise. VLM training is the same stack with two visual sub-stages bolted on.

## Prerequisites

Module 01 (decoder-only LMs, KV cache).

## The map

```
   1. Pre-training              web text, causal LM       "base" model
        ↓
   2. SFT (instruction tuning)  (prompt, response) pairs  follows instructions
        ↓
   3. Preference alignment      (prompt, chosen,          helpful + on-style
      (RLHF / DPO / GRPO)        rejected) triples
        ↓
   4. (optional) Reasoning RL   verifiable rewards        R1-style reasoning
```

VLM-specific sub-stages between 1 and 2:

```
   1.5  Vision-language alignment    train only the projector;
                                     image-caption pairs (frozen LLM + encoder)
   2.5  Visual instruction tuning    train projector + LLM;
                                     GPT-4-generated image-conversations
```

### 2. Pre-training and SFT

Pre-training is what you already know from Module 01: next-token cross-entropy on a huge corpus. Produces a "base" model that completes text but doesn't follow instructions.

**SFT** continues training the base model on a small high-quality (prompt, ideal-response) dataset — same loss, just on curated data. Cheapest stage, biggest behavioral change. The dataset *choice* matters more than the loss.

**Visual instruction tuning** (LLaVA's contribution, Module 03) is just SFT with image-conversation pairs generated by GPT-4 from captions. Read LLaVA as much for the data trick as the architecture.

- **Primary (covers SFT + RLHF):** [Hugging Face — *Illustrating RLHF*](https://huggingface.co/blog/rlhf) — first half is SFT, second half is RLHF; diagrams throughout.

### 3. The three preference-alignment recipes side-by-side

| | **PPO-RLHF** (classical) | **DPO** (modern default) | **GRPO** (reasoning-RL) |
|---|---|---|---|
| **Training data** | `(prompt, chosen, rejected)` → trains reward model first | `(prompt, chosen, rejected)` directly | `(prompt, K samples, rewards)` — verifiable or learned |
| **Extra models?** | Yes: reward model + reference model + value-function critic | No — closed-form loss on the SFT model + reference | No critic — uses **group baseline** (avg reward over K samples) |
| **Loss type** | RL (policy gradient + KL penalty) | Single classification-style loss | RL with group-normalized advantage |
| **Stability** | Finicky | Stable, easy | Stable; scales well |
| **Famous example** | InstructGPT (1.3B PPO'd beat GPT-3 175B) | Most open-source models today | DeepSeek-R1 reasoning model |
| **Read it?** | No — HF blog above is enough. | Yes — [DPO paper](https://arxiv.org/abs/2305.18290). **Skim path:** §4 + §6. | Skim — [DeepSeekMath](https://arxiv.org/abs/2402.03300). **Skim path:** §4.1 only. |

For VLMs, the same recipes run on image-grounded preference data (chosen vs. rejected response to the same image+prompt). **Qwen2.5-VL, InternVL3, and most newer open VLMs use DPO or descendants.**

**Cross-link to Module 07:** VideoBrain uses a behavior-aware reward to shape a frame-selection policy — same family as GRPO with verifiable rewards.

If you only remember one frame: **after pre-training, every modern model is a small data trick on top of "SFT + (DPO or GRPO)". The rest is variations.**

## Self-check

- **Name the four stages and one example of each stage's training data.**
- **Why is DPO simpler than PPO-RLHF?** (No reward model, no value function, no RL roll-outs. One supervised-style loss on preference triples.)
- **What's a "base" model vs. an "instruct" model?** (Base = stage 1 only. Instruct = stage 2+; usually labeled `…-Instruct` or `…-Chat`.)
- **For VLMs, what does stage 1.5 train and what's frozen?** (Only the projector. LLM + vision encoder are frozen. Data: image-caption pairs.)
